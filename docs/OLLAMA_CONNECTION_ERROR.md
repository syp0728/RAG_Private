# Ollama 연결 오류 해결

## 오류 메시지
```
[WinError 10061] 대상 컴퓨터에서 연결을 거부했으므로 연결하지 못했습니다
```

이것은 **Ollama 서버가 실행되지 않았거나** 연결할 수 없는 상태입니다.

## 해결 방법

### 1단계: Ollama 서버 실행 확인

**새 PowerShell 터미널**을 열고:

```powershell
# Ollama 서버가 실행 중인지 확인
netstat -ano | findstr :11434
```

**결과가 없으면** Ollama 서버가 실행되지 않은 것입니다.

### 2단계: Ollama 서버 시작

**새 PowerShell 터미널**에서:

```powershell
ollama serve
```

서버가 시작되면 다음과 같은 로그가 표시됩니다:
```
Listening on 127.0.0.1:11434 (version 0.13.5)
```

**중요**: 이 터미널을 닫지 마세요! Ollama 서버가 계속 실행되어야 합니다.

### 3단계: 백엔드 재시작 (선택사항)

Ollama 서버를 시작한 후, 백엔드 서버를 재시작하면 자동으로 연결됩니다.

## 확인 방법

Ollama 서버가 정상적으로 실행 중인지 확인:

```powershell
# 다른 터미널에서
ollama list
```

모델 목록이 나오면 정상입니다.

## 자동 시작 설정 (선택사항)

Ollama를 항상 실행되도록 하려면:

1. **Windows 작업 스케줄러** 사용
2. 또는 **시작 프로그램**에 추가

하지만 가장 간단한 방법은:
- Ollama 서버를 별도 터미널에서 실행
- 백엔드 서버를 다른 터미널에서 실행

## 정상 작동 시 로그

Ollama 서버가 정상적으로 실행 중이면:

```
[RAG] Ollama 연결 시도: localhost:11434, 모델: llama3.1:8b-instruct-q4_K_M
[RAG] Ollama 응답 받음 (길이: XXX자, 소요 시간: X.XX초)
```

## 요약

1. **새 터미널 열기**
2. **`ollama serve` 실행**
3. **터미널을 열어둔 상태로 유지**
4. **RAG 시스템에서 질문 다시 시도**

Ollama 서버가 실행 중이면 정상적으로 작동합니다!
